# Starless 多尺度结构增强设计

日期：2026-06-19

## 1. 目标

为 `deep-sky-processor` 增加一条面向已拉伸 starless 图像的确定性后期流程。在不生成新纹理、不补画结构的前提下，按天体类型突出原图中已有的：

- 星云气体分布、褶皱、壳层和细丝；
- 反射尘埃流与暗云层次；
- 星系旋臂、外围盘、尘埃带和 HII 小结；
- 行星状星云内外壳、环状边缘和喷流；
- 超新星遗迹的长丝和冲击前沿；
- 宽场星云及银河暗尘网络。

流程必须使用独立的 StarNet stars 图层完成最终重组，并输出 LOW、MEDIUM、HIGH 三档候选。默认审美目标偏向强烈视觉冲击，但任何档位都不能绕过真实性和质量门禁。

## 2. 范围

### 2.1 首版包含

- 输入已拉伸的 TIFF 或 PNG starless 图像。
- 输入与 starless 完全配准的独立 stars 图层。
- 要求显式传入 `target_type`。
- 支持当前技能中的非星团目标类型：
  - `emission_nebula`
  - `reflection_nebula`
  - `dark_nebula`
  - `molecular_cloud`
  - `galaxy` 及星系子类
  - `planetary_nebula`
  - `supernova_remnant`
  - `wide_field`
  - 其他无法匹配专用策略的非星团类型使用保守通用策略
- 输出三档 starless 候选和三档最终带星合成图。
- 输出蒙版、尺度细节、差异图和统一 JSON 审查报告。

### 2.2 首版不包含

- 线性 FITS/XISF 的 starless 美化。
- 自动视觉识别或 CV 推断天体类型。
- 从带星图减去 starless 图推导 stars 图层。
- 训练模型、生成式补纹理、AI 着色或在线服务。
- 对主管线做无关重构。

### 2.3 必须拒绝的目标

球状星团、疏散星团、一般星团以及 M45 等星点属于主体的目标，入口必须拒绝 starless 流程，并提示使用原带星处理流程。质量门禁不能用于绕过此规则。

## 3. 输入契约

新入口接收：

1. `starless_input`：已拉伸的 RGB TIFF 或 PNG。
2. `stars_input`：StarNet 独立输出的 stars 图层。
3. `target_type`：显式天体类型。
4. 可选的输出目录、母版格式和强度限制。

输入校验要求：

- 两个图层宽高、通道数、方向和裁切完全一致。
- 图像必须为有限数值，归一化后位于可处理范围。
- stars 图层应以黑背景承载正向星点信号。
- 对明显不合理的 stars 图层，例如大面积非黑背景、负值残差、形状不匹配，直接失败。
- 不提供“原图减 starless”的自动回退。

## 4. 总体架构

建议新增独立的多尺度 starless 增强模块，避免继续扩大 `compose_starnet_layers.py` 的职责。现有脚本保留为 CLI 编排和兼容入口。

处理数据流：

```text
输入校验
→ starless 诊断
→ target_type 策略选择
→ 主体 / 暗结构 / 保护蒙版
→ 大 / 中 / 小尺度结构分解
→ 受限结构增强和融合
→ LOW / MEDIUM / HIGH 三档 starless 候选
→ 每档独立 stars 处理
→ 星点重组
→ 数值门禁、自动收缩和视觉审查包
```

### 4.1 组件边界

建议组件如下：

- `starless_diagnostics`
  - 计算亮度分布、动态范围、噪声、色彩健康、背景均匀度、边缘能量和尺度结构能量。
  - 不修改图像。
- `starless_target_profiles`
  - 将 `target_type` 映射为结构目标、尺度权重、色彩限制和参数上限。
  - 只提供策略，不直接处理像素。
- `starless_masks`
  - 从输入图构建主体、暗结构、高光、背景和低信噪保护蒙版。
  - 输出可保存、可审查的浮点软蒙版。
- `starless_multiscale`
  - 分解大、中、小尺度结构。
  - 只从输入图已有亮度和色彩变化产生 detail 层。
- `starless_enhancer`
  - 根据 profile 和诊断结果增强各尺度，并通过蒙版受限融合。
  - 生成 LOW、MEDIUM、HIGH 三档候选。
- `stellar_recomposer`
  - 对同一个 stars 输入生成三档星点层并完成重组。
- `starless_quality`
  - 比较输入、starless 候选和最终合成图，执行门禁和降级。

各组件通过 NumPy 浮点 RGB 图像、蒙版和 JSON-safe 报告通信，避免隐藏的全局状态。

## 5. 诊断与蒙版

### 5.1 Starless 诊断

诊断至少包括：

- 亮度中位数、低端和高端分位数；
- 非正像素、接近黑位像素和通道裁切比例；
- 暗部、主体、中亮区和高光区的局部噪声估计；
- 角落和网格背景均匀度；
- 色彩饱和度分布、通道不平衡和背景彩噪；
- 多尺度高频能量；
- 方向性边缘和长结构连续性参考；
- 疑似去星伪影区域，例如小型暗坑、亮环和局部双边。

诊断用于收缩 profile 参数，不得自动把 HIGH 档推到超出安全上限。

### 5.2 蒙版类型

每次处理生成三类逻辑蒙版：

- 主体蒙版：
  - 选择高于背景、具有稳定结构能量的真实信号。
  - 控制大尺度塑形和中尺度细节增强。
- 暗结构蒙版：
  - 选择邻域中相对更暗且具有连续边界的结构。
  - 用于尘埃带和暗云，不直接执行全局压黑。
- 保护蒙版：
  - 联合背景、低信噪、高光、通道裁切和疑似去星伪影区域。
  - 对增强增益做乘法抑制。

所有蒙版必须软化并输出覆盖率。近空蒙版、覆盖过宽或硬边明显时，候选进入审查或失败。

## 6. 多尺度结构增强

### 6.1 尺度职责

- 大尺度：
  - 主体与背景的分离；
  - 星云整体气体分布；
  - 星系外围盘；
  - 宽场云气分区。
- 中尺度：
  - 星云褶皱和壳层；
  - 星系旋臂和尘埃带；
  - 暗云层叠关系；
  - 行星状星云内外壳。
- 小尺度：
  - 星云细丝；
  - 尘埃边界；
  - HII 小结；
  - 超新星遗迹冲击前沿；
  - 小型喷流和环状纹理。

实际尺度应根据图像尺寸和诊断出的结构能量计算，不能只使用固定像素半径。

### 6.2 融合原则

- detail 层必须由原图与平滑层的确定性差异产生。
- 正向亮结构增强与暗结构层次增强使用不同路径。
- 暗结构通过邻域亮度分离、边界对比和周围主体塑形表现，不把暗区直接压成黑洞。
- 低信噪背景不得接受小尺度增强。
- 高光区仅允许受限 rolloff 和中尺度层次恢复。
- 色彩增强跟随已有色度，不能从亮度结构推导不存在的颜色。
- 每个尺度的增益有独立上限，并受保护蒙版约束。

## 7. 按天体类型的策略

### 7.1 发射星云

- 突出大尺度气体、中尺度褶皱、壳层和细丝。
- 中尺度权重最高，小尺度保持受控。
- 保护亮核和低信噪背景。
- 保持 Hα/OIII 既有比例，避免全图饱和导致品红或电蓝。

### 7.2 反射星云

- 突出柔和尘埃流、反射边缘和亮度渐变。
- 大、中尺度为主，小尺度权重较低。
- 蓝色饱和度设置更严格上限，避免尘埃变脆或电蓝。

### 7.3 暗星云与分子云

- 突出暗云轮廓、遮挡层次和尘埃边界。
- 暗结构蒙版权重最高。
- 禁止通过大幅降低暗部亮度制造对比。
- 优先增强暗结构周围已有的亮背景和连续边缘。

### 7.4 星系

- 突出旋臂连续性、尘埃带、外围盘和 HII 小结。
- 核心使用高光保护和 rolloff。
- 方向性增强沿连续结构工作，避免旋臂碎裂或出现颗粒状锐化。
- 外围弱信号不接受激进降噪或黑位压缩。

### 7.5 行星状星云

- 突出内外壳、环状边缘、喷流和小尺度纹理。
- 中、小尺度允许较高权重。
- 保护中心亮核及既有 OIII 色彩。

### 7.6 超新星遗迹

- 突出长丝、弧状壳层和冲击前沿。
- 连续性门禁严格高于其他目标。
- 禁止将噪声点或去星残留连接成假细丝。

### 7.7 宽场星云与银河

- 突出大尺度云气分区和暗尘网络。
- 大尺度权重最高，小尺度权重最低。
- 背景均匀度门禁更严格，避免整幅图颗粒化。

### 7.8 通用非星团目标

无法匹配专用 profile 时使用保守通用策略：

- 大、中尺度轻量增强；
- 小尺度默认关闭或保持极低权重；
- 不做专用色相增强；
- 输出状态至少为 `review_required`，除非视觉审查确认。

## 8. LOW、MEDIUM、HIGH 候选

三档共享同一组诊断和基础蒙版，但使用不同的受限参数。

### 8.1 LOW

- 结构增益约为 profile 基准的 `0.55×`。
- 色彩和主体塑形轻量。
- stars 初始合成强度约 `0.90`。
- 星点只做轻微软化和保守饱和度调整。

### 8.2 MEDIUM

- 结构增益使用 profile 基准 `1.00×`。
- 色彩和主体塑形为平衡级别。
- stars 初始合成强度约 `0.78`。
- 使用标准柔化和适度星色保护。

### 8.3 HIGH

- 结构增益初值约为 profile 基准的 `1.35×`，但受硬上限和诊断收缩约束。
- 允许更强大尺度塑形、中尺度局部对比和受控饱和度。
- stars 初始合成强度约 `0.66`。
- 星点柔化略强，但不能改变星点位置、连通关系和基本形态。

这些数值是 profile 初值，不是无条件硬编码值。高噪声、低动态范围、严重去星残留或高光裁切会收缩实际参数。

## 9. Stars 图层处理与重组

三个候选使用同一个 stars 输入，但分别处理：

- 亮度强度；
- 色彩饱和度；
- 轻微软化；
- 合成强度。

约束如下：

- 不移动、补充或删除星点。
- 不通过形态学操作显著改变星点拓扑。
- 不把所有恒星去色为白色。
- 不产生黑环、双星、硬边或明显膨胀。
- 重组后不得重新淹没主体。
- 优先沿用现有正向加法重组方式；若未来增加 screen/unscreen 模式，需另行设计和验证。

## 10. 质量门禁与自动降级

### 10.1 Starless 候选门禁

- 黑位和非正像素比例不能异常增加。
- 暗部裁切比例不能超过 10.4 节定义的基准。
- 角落及网格背景均匀度不能超过 10.4 节定义的恶化幅度。
- 低信噪区域高频能量增幅受限。
- 结构边缘不得出现明显亮环、暗环或双边。
- 长丝、旋臂和尘埃边界连续性不能显著下降。
- 通道裁切、色相漂移和背景彩噪受限。
- 增强差异必须主要落在主体或暗结构蒙版内。

### 10.2 重组后门禁

- 星点峰值和高光裁切比例受限。
- 星点面积比例不能异常上升。
- 星色分布不能塌缩为大面积白色或单一偏色。
- 星点周围不得出现黑环、双边或硬边。
- 主体与背景对比不能被 stars 图层完全抵消。
- 最终亮度、饱和度和背景范围符合目标 profile 的安全区间。

### 10.3 降级策略

每档第一次失败时，仅收缩触发失败的维度，例如：

- 光晕失败：降低对应尺度增益；
- 噪声失败：降低小尺度权重并加强低信噪保护；
- 黑位失败：减弱大尺度塑形和黑位调整；
- 色彩失败：降低饱和度或色彩分离；
- 星点失败：提高 stars 强度安全值、降低柔化或饱和调整。

每档最多自动重试一次。第二次仍失败时：

- 不得标记为成功；
- LOW 或 MEDIUM 可保留其他已经通过的候选；
- HIGH 标记为 `review_required`，严重失败时不输出最终合成图；
- 报告必须记录原参数、降级参数和失败门禁。

### 10.4 首版门禁基准

首版使用相对于输入基线的保守阈值，避免不同实现者自行解释“显著恶化”：

- 接近黑位像素比例的绝对增量不得超过 `2%`。
- 任一 RGB 通道高光裁切比例的绝对增量不得超过 `0.5%`。
- 角落均匀度指标不得比输入恶化超过 `20%`。
- 保护蒙版内的高频能量不得比输入增加超过：
  - LOW：`8%`
  - MEDIUM：`15%`
  - HIGH：`25%`
- 至少 `70%` 的绝对增强差异能量必须位于主体蒙版或暗结构蒙版内。
- 主体蒙版软覆盖率低于 `0.5%` 时失败；高于 `85%` 时进入 `review_required`。
- 星点峰值裁切比例不得比 stars 输入增加超过 `0.5%`。
- 重组后的星点面积比例不得比按原强度直接重组的基准增加超过 `10%`。
- 结构连续性指标相对输入下降超过 `10%` 时失败；该门禁适用于星系、超新星遗迹和具有明显壳层的目标。

阈值需要通过真实样本回归校准。校准只能收紧默认值，或在设计文档更新并完成回归验证后调整；运行时不得为了让 HIGH 档通过而动态放宽。

## 11. 输出契约

建议输出：

```text
starless_low.tif
starless_medium.tif
starless_high.tif

final_low.tif
final_low.jpg
final_medium.tif
final_medium.jpg
final_high.tif
final_high.jpg

review/
  subject_mask.*
  dark_structure_mask.*
  protection_mask.*
  detail_large.*
  detail_medium.*
  detail_small.*
  low_before_after_difference.*
  medium_before_after_difference.*
  high_before_after_difference.*

report.json
```

仅对通过或允许人工审查的候选生成对应最终文件。统一报告至少包含：

- 输入文件和校验结果；
- `target_type` 和所选 profile；
- starless 诊断；
- 蒙版统计；
- 每档实际尺度参数；
- 每档 stars 参数；
- 每项质量门禁；
- 自动降级记录；
- 最终状态；
- 按天体类型生成的视觉审查清单。

## 12. CLI 与现有代码集成

首版建议新增独立 CLI，例如：

```bash
python scripts/enhance_starless.py \
  starless.tif stars.tif output_dir \
  --target-type emission_nebula \
  --master-format tif \
  --report output_dir/report.json
```

现有 `compose_starnet_layers.py` 可在后续改为调用新模块，保留原参数兼容行为。主管线 `pipeline.py` 首版只需要在文档中推荐新入口，不要求立即重写 starless 分支。

## 13. 错误处理

以下情况立即失败：

- 输入不存在或格式不支持；
- 图像尺寸、通道或方向不匹配；
- 缺少 `target_type`；
- 缺少独立 stars 图层；
- 目标属于拒绝列表；
- 图像包含大量非有限值且无法安全清理；
- stars 图层明显不是黑背景正向星点层。

以下情况允许生成审查产物但状态为 `review_required`：

- 通用非星团 profile；
- 蒙版覆盖异常但仍有可解释结构；
- HIGH 档接近但未触发硬失败阈值；
- 疑似去星伪影较多；
- 数值门禁通过但必须进行人工视觉判断的强增强。

## 14. 测试设计

### 14.1 单元测试

- 输入和目标类型校验。
- 星团及 M45 拒绝规则。
- 自适应尺度随图像尺寸变化。
- 主体、暗结构和保护蒙版覆盖率。
- 暗结构增强不会直接系统性压低暗区。
- 低信噪区不会接受高强度小尺度增强。
- LOW、MEDIUM、HIGH 参数单调但不突破上限。
- stars 图层处理不移动星点位置或改变主要连通关系。
- 失败维度只收缩对应参数。

### 14.2 合成图测试

构造可控样本：

- 发射星云壳层和细丝；
- 柔和反射尘埃；
- 暗云遮挡带；
- 星系核心、旋臂和尘埃带；
- 行星状星云环壳；
- 超新星遗迹长丝；
- 宽场大尺度云气。

验证增强差异主要落在预期蒙版和尺度上，并检查不存在的新结构。

### 14.3 回归测试

- 现有 `beautify_starless` 行为保持兼容。
- 现有 StarNet stars 输入和重组测试继续通过。
- `style_tools`、`mask_tools` 和 `quality_metrics` 不被破坏。
- 主管线星团安全规则保持有效。

### 14.4 视觉验收

每种支持的目标类型至少使用一张真实样本比较三档结果。重点检查：

- 主体是否更清楚但未生成假纹理；
- 气体和尘埃是否保持连续、自然；
- 暗云是否有层次而不是黑洞；
- 旋臂是否连续且未碎裂；
- HIGH 档是否出现光晕、断层、颗粒化或塑料感；
- stars 是否自然且不重新压制主体。

## 15. 成功标准

实现满足以下条件时可认为首版完成：

- 对所有支持的 `target_type` 生成三档候选或明确报告失败原因；
- 输入和拒绝规则不可绕过；
- 三档主体增强和 stars 权重呈明确梯度；
- 数值差异主要位于真实结构蒙版内；
- HIGH 档失败时能够自动收缩或诚实降级；
- 不使用生成式处理，不产生输入中不存在的结构和颜色；
- 回归测试通过，真实样本完成视觉审查。
