# 线性阶段预处理

本文覆盖拉伸前容易遗漏的四个环节：坏点修复、子帧局部归一化、梯度类型诊断和宽场梯度处理。目标是先得到统计一致、背景可信的线性母版，再进入校色、降噪、去星和拉伸。

## 能力边界

- 坏点修复和局部归一化应在**子帧级**完成，早于最终堆栈。
- 当前 `pipeline.py` 接收叠加后的母版，不替代校准、配准、Cosmetic Correction 或 Local Normalization。
- `analyze.py` 可以诊断叠加母版的低频梯度并推荐 DBE 方法。
- `gradient_removal.py` 和管线内置 DBE 只用于叠加后的背景修正，不能补救错误的暗场、平场或严重云层污染。
- 所有线性中间文件优先使用 float32 FITS/XISF/TIFF，避免过早裁剪负值或量化为 16-bit。

推荐顺序：

```text
主偏置/暗场/平场校准
→ 坏点修复
→ 子帧质量筛选
→ 配准
→ 局部归一化
→ 带拒绝算法的堆栈
→ 梯度诊断
→ 必要时执行宽场背景建模
→ 线性降噪、校色、去星
→ 拉伸
```

## 1. 坏点修复

### 适用对象

- 热像素：位置固定、亮度显著高于邻域。
- 冷点/死点：位置固定、响应显著低于邻域。
- 暗场无法完全修正的残余坏点。
- 单帧宇宙射线不属于固定坏点，优先由堆栈拒绝算法处理。

### 推荐方法

优先级从高到低：

1. 使用与温度、增益、曝光时间匹配的主暗场校准。
2. 从主暗场和主平场生成 defect map / cosmetic correction map。
3. 仅替换已确认的异常像素，取周围正常像素的中位数或双线性插值。
4. 堆栈时使用 Winsorized Sigma Clipping 等拒绝算法清理随机异常。

不要对整张图使用强中值滤波代替坏点修复。它会改变星点 FWHM、削弱微小恒星并破坏窄带细丝。

### 阈值原则

坏点必须同时满足：

- 相对局部邻域显著异常；
- 在多张子帧中位置稳定；
- 不符合恒星点扩散函数；
- 不沿配准后的天空坐标移动。

只在单帧出现的亮点应标记为瞬态异常，交给堆栈拒绝；跨帧固定在传感器坐标上的异常才进入 defect map。

### 工具建议

- PixInsight：`CosmeticCorrection`，优先使用主暗场生成的 defect list。
- Siril：在校准序列上执行 cosmetic correction，并检查修正像素数量。
- AstroPixelProcessor / DeepSkyStacker：使用坏点图和 sigma rejection，避免启用会模糊星点的全局滤波。

### 质量检查

- 修复前后星点数量和 FWHM 不应系统性下降。
- 暗背景中不再出现固定位置的红、绿、蓝孤立点。
- 修复图与原图差分应主要由稀疏单像素构成。
- 若差分出现大量星核或细丝结构，阈值过宽，应回退。

## 2. 局部归一化

局部归一化用于统一不同夜晚、月相、透明度、气辉和光害条件下的子帧背景与响应。它不是简单地把每张图的中位数缩放到相同值。

### 使用条件

建议启用：

- 多夜数据；
- 子帧背景中位数、颜色或梯度方向明显变化；
- 薄云造成缓慢空间变化；
- 子帧曝光时间或滤镜批次不同，但信号仍可可靠标定。

通常可省略：

- 单夜、同曝光、透明度稳定；
- 平场和背景高度一致；
- 子帧数量很少，局部模型容易受星云结构污染。

### 参考帧选择

参考帧应满足：

- 天空透明度高；
- FWHM 小、偏心率低；
- 背景较低且无云；
- 没有明显卫星轨迹、反射或局部漏光；
- 构图覆盖完整。

不要单纯选择背景最暗的一帧。若它透明度差，归一化会错误放大噪声。

### 处理原则

局部模型应只描述大尺度响应差异：

```text
normalized(x, y) =
    scale(x, y) × subframe(x, y) + offset(x, y)
```

- 模型尺度必须远大于星点和目标细节。
- 背景样本应排除亮星、星云主体、暗尘埃和图像边缘。
- 对 Hα 宽场或大尺度 IFN，优先使用保守的大网格，避免把真实弥散结构归一化掉。
- 不同滤镜分别归一化，不能用 RGB 或窄带通道互相作为参考帧。

### 异常帧处理

局部归一化不是挽救坏帧的工具。出现以下情况时优先剔除子帧：

- 云层造成非平滑遮挡；
- 梯度模型与参考帧差异过大；
- 星点通量缩放异常；
- 大片区域缺失或边缘配准不足；
- 归一化后噪声被显著放大。

### 质量检查

堆栈前检查：

- 各子帧背景中位数和 MAD 收敛；
- 恒星通量比例保持稳定；
- 归一化模型不包含可辨认的星云、尘埃或星点轮廓；
- 多夜数据在接缝区域没有亮度台阶。

堆栈后检查：

- 背景不出现局部色块或网格印；
- 暗弱目标没有被削平；
- 通道间信号比例仍符合滤镜响应和曝光时间。

## 3. 梯度类型诊断

先诊断，再选择背景模型。运行：

```bash
python scripts/analyze.py stacked_linear.fits \
  --output analysis.json \
  --format readable
```

重点读取：

- `gradient.gradient_pattern`
- `gradient.gradient_severity`
- `gradient.gradient_x` / `gradient.gradient_y`
- `gradient.r_squared`
- `gradient.vignetting_ratio`
- `gradient.channel_gradients`
- `gradient.chromatic_gradient_review`
- `gradient.dbe_decision`

### 类型判定

| 模式 | 数值/视觉特征 | 常见原因 | 建议 |
|---|---|---|---|
| `none` | 方向梯度弱，角落与中心接近 | 校准良好或真实均匀背景 | 跳过 DBE |
| `linear` | 单一方向缓变，平面拟合 R² 高 | 城市光害、月光、地平线气辉 | 低阶 polynomial |
| `vignetting_dominant` | 中心亮、四角对称变暗，`vignetting_ratio` 低 | 平场不足、镜头渐晕 | 先检查平场，再做宽场模型 |
| `complex` | 平面拟合 R² 低，存在多个低频区域 | 多光源、反射、复杂气辉 | 保守 RBF 或外部 DBE |
| `irregular` | 无稳定方向，残差较大 | 噪声、薄云、校准缺陷 | 先排查坏帧；必要时 median |

### 方向性梯度与均匀偏置

方向性梯度表现为一侧持续变亮，`gradient_x` 或 `gradient_y` 有稳定符号，通常需要减去空间模型。

均匀偏置表现为全场背景整体偏高但空间变化很小。它不应通过高阶 DBE 修正：

- 线性母版可保留统一背景基线；
- 后续黑位和拉伸阶段再决定显示背景；
- 若来自偏置/暗场校准错误，应回到校准阶段修复。

### 色度梯度

灰度梯度可能因 RGB 通道互相抵消而显示为 `none`。当：

```text
dbe_decision == review_chromatic
```

必须暂停自动 DBE，分别查看 R/G/B 低频模型和差分。优先区分：

- 真实 Hα、OIII、IFN 或银河背景；
- 滤镜反射、月光和通道相关光污染；
- 平场色偏或传感器响应差异。

不要为了得到灰色背景而抹除真实的大尺度发射结构。

## 4. 宽场梯度专用处理

宽场图像常同时包含镜头渐晕、传感器阴影、平场残差、地平光害和真实银河/星云结构。单次高阶全图拟合风险很高。

### 第一步：排除校准问题

若背景模型呈现以下形状，优先重做平场而不是 DBE：

- 明显的灰尘环；
- 镜头遮光罩或滤镜边缘阴影；
- 传感器读出带状结构；
- 与光学中心严格同心的强渐晕；
- 不同夜晚旋转后仍固定在传感器坐标的缺陷。

DBE 只能减弱平滑残差，不能可靠修复结构化平场错误。

### 第二步：分层建模

建议把模型拆成两类：

1. 径向低频层：处理大型渐晕，使用非常平滑的曲面。
2. 方向低频层：处理地平线、月光或城市光害，使用低阶平面/多项式。

每层都使用保守强度，并在相减后检查背景模型是否包含真实目标。避免一次使用过高阶模型同时解决全部问题。

### 方法选择

- 单一方向光害：`polynomial`, degree 1–2。
- 轻度渐晕叠加方向梯度：`polynomial`, degree 2–3。
- 多区域复杂光害：外部手工 DBE 优先；自动处理可试 `rbf`，但必须审查。
- 没有清晰空间结构、主要是噪声：跳过，或谨慎使用大尺度 `median`。

管线示例：

```bash
python scripts/pipeline.py stacked_linear.fits widefield.jpg \
  --strength adaptive \
  --target-type wide_field \
  --steps dbe,color,pre_denoise,stretch,final_color \
  --keep-all
```

明确覆盖诊断建议时：

```bash
python scripts/pipeline.py stacked_linear.fits widefield.jpg \
  --strength adaptive \
  --target-type wide_field \
  --override-params '{"dbe_method":"polynomial","dbe_degree":2}' \
  --keep-all
```

### 样本布置

- 避开银河、IFN、大片 Hα、反射星云和暗尘埃。
- 避开亮星光晕、反射鬼影、堆栈边缘和黑边。
- 全场均匀分布样本，角落不能缺失。
- 样本尺度应大于恒星和小型星云，但小于待建模的梯度尺度。
- 目标覆盖画面大部分时，应减少自动样本并改用人工背景区域。

### 背景模型审查

必须同时检查：

1. 原始线性图的安全预览。
2. 估计出的背景模型。
3. 校正后图。
4. 原图减校正图的差分。

背景模型只应包含非常平滑的低频结构。若能看到银河尘带、星云壳层、彗尾或恒星轮廓，说明模型污染，必须回退。

### 通过标准

- 四角不再有明显亮度台阶，但不强求数学上完全一致。
- 中心没有被挖出黑坑。
- 大尺度真实结构在差分前后保持连续。
- RGB 通道没有新增彩色边缘或局部偏色。
- 拉伸后看不到网格、波纹、环状反转或边缘过减。

## 失败回退

| 症状 | 可能原因 | 回退措施 |
|---|---|---|
| 星点变钝、微星消失 | 坏点阈值过宽 | 缩小 defect map，仅修固定单像素 |
| 堆栈出现网格或色块 | 局部归一化尺度过小 | 增大模型尺度，重新选择参考帧 |
| 星云外围被削弱 | 背景样本落入真实结构 | 移除污染样本，降低模型自由度 |
| 中心出现黑坑 | 渐晕模型过减 | 降低阶数/强度，优先重做平场 |
| RBF 出现波纹 | 样本不足或过拟合 | 改用低阶 polynomial 或外部手工 DBE |
| 各通道梯度方向不同 | 色度梯度或平场问题 | 分通道检查，禁止自动强制中和 |

## 最小交付记录

处理多夜或宽场数据时，至少记录：

- 校准库的温度、增益、曝光和日期；
- 坏点图来源与修复像素数量；
- 局部归一化参考帧；
- 被剔除的异常子帧及原因；
- `gradient_pattern`、`vignetting_ratio` 和 DBE 决策；
- 背景模型及差分审查结论。
