# 特殊目标处理

本文覆盖彗星、行星、月面、宽场星野和超新星残骸。它们不完全符合普通深空目标的处理假设，必须先完成各自的专用校准、配准或合成，再按需接入 `pipeline.py`。

## 总体能力边界

- 当前主管线原生支持 `wide_field`。
- 风格模块可识别 `comet` 和 `supernova_remnant`，但主管线 `--target-type` 尚未暴露这两个值。
- 彗星对齐、行星幸运成像、月面视频堆叠和全景拼接不是当前管线能力。
- 外部流程应输出 16-bit 或 float32 TIFF/FITS 母版，再进入深空管线做有限的色调、动态范围和降噪处理。
- 不使用生成式补帧、AI 超分辨率、纹理重建或不存在的色彩。

## 1. 彗星

### 数据特点

- 彗核相对恒星持续移动。
- 恒星配准会拖曳彗星，彗核配准会拖曳恒星。
- 彗发动态范围高，彗尾低表面亮度且容易被背景模型误减。
- 离子尾可能快速变化，不同时间段不能无条件叠加。

### 拍摄与分组

- 记录每帧 UTC 时间、曝光、滤镜和彗星位置。
- 时间跨度较长时按彗尾形态和彗核位移分组。
- RGB 数据应保持各通道时间接近，避免彗核移动产生彩边。
- 彗核已经饱和时，后期无法恢复核心结构；应补拍短曝光。

### 双重对齐流程

```text
校准子帧
→ 星点配准序列
→ 彗核配准序列
→ 分别质量筛选与堆栈
→ 星点层/彗星层分离
→ 确定性合成
→ 动态范围和颜色处理
```

#### 恒星配准堆栈

用于得到圆形、静止的背景恒星。堆栈拒绝应尽量消除移动彗星残影，但不能依赖过强 sigma clipping 擦除真实星云背景。

#### 彗核配准堆栈

以彗核亮度中心或人工测量位置对齐。目标是保持彗发和彗尾连续。若彗尾在时间上变化明显，应缩短合成时间窗。

#### 最终合成

- 从恒星配准结果提取背景星场。
- 从彗核配准结果提取彗星层。
- 使用真实遮罩合成“静止恒星 + 静止彗星”。
- 检查彗核周围是否出现空洞、重复核心、星点双影或接缝。

不要用模糊、克隆或生成式填补伪造彗尾结构。

### 彗尾处理

- DBE 样本必须避开整条彗尾和可能的反尾。
- 优先在单帧或恒星配准堆栈中判断背景方向，不能把长直彗尾当作光害梯度。
- 局部对比只作用于彗尾有效信号区域，强度应低。
- 离子尾通常偏蓝，尘埃尾接近中性或暖色；不要统一染色。
- 降噪应使用亮度蒙版保护彗尾细丝，避免塑料化。

### 动态范围

彗核、彗发和彗尾的亮度跨度很大。推荐：

1. 用短曝光保护彗核。
2. 用长曝光记录彗尾。
3. 分组校准、配准并堆栈。
4. 按真实曝光响应做 HDR 合成。
5. 最终使用高光保护拉伸。

只有单一曝光时，可以压缩未饱和高光，但不能宣称恢复饱和彗核。

### 管线接入

先在 Siril、PixInsight 等工具完成彗星合成，再使用通用轻度步骤。不要传入主管线不支持的 `comet` 类型：

```bash
python scripts/pipeline.py comet_combined.tif comet_final.jpg \
  --strength light \
  --steps stretch,final_color,sharpen,final_denoise \
  --style natural \
  --keep-all
```

默认跳过：

- 自动 DBE：可能减掉彗尾。
- `star_remove` / `star_reduce`：可能破坏彗核或合成后的星场。
- 强局部增强：容易制造尾部断层。

### 质量检查

- 彗核只有一个亮度中心。
- 星点无双影、拖线或彗星形空洞。
- 彗尾方向和分叉与原始帧一致。
- 背景模型中看不到彗尾。
- 高光无硬剪切，彗核周围无黑圈。

## 2. 行星

### 数据特点

- 通常来自高速视频或大量短曝光。
- 清晰度主要由瞬时视宁度决定。
- 目标小、亮度高，自转可能在短时间内改变表面结构位置。
- RGB/滤镜轮拍会产生通道时间差和大气色散。

### 幸运成像流程

```text
视频预处理
→ 稳定目标位置
→ 质量评估和排序
→ 选择最佳帧
→ 表面对齐与多点对齐
→ 堆栈
→ RGB 通道对齐
→ 必要时去旋转
→ 反卷积/波形锐化
→ 轻度降噪和色彩校准
```

推荐工具：

- PIPP：视频裁切、居中、格式转换和初筛。
- AutoStakkert：质量排序、多点对齐和幸运成像堆栈。
- WinJUPOS：木星等快速自转目标的图像/视频去旋转。
- RegiStax、AstroSurface：波形锐化、RGB 对齐和轻度降噪。
- PixInsight：反卷积、色彩和最终合成。

### 帧选择

- 选择比例由视宁度决定，不固定追求 5% 或 10%。
- 好视宁度可提高入选比例；差视宁度应减少比例。
- 比较多个堆栈版本，例如最佳 5%、10%、25%、50%。
- 入选帧增加但细节下降时，应选择更低比例。

### RGB 通道对齐

需要分别处理：

- 通道整体平移：校正拍摄或软件配准偏差。
- 行星边缘红蓝分离：优先使用 ADC 拍摄，后期只做残余校正。
- 单色相机轮拍：按拍摄中间时间对齐，并考虑目标自转。
- 不同通道分辨率差异：以真实较差通道为限制，不能强行补造细节。

### 锐化

推荐顺序：

1. 在高位深线性或近线性堆栈上做反卷积/波形锐化。
2. 使用行星圆盘蒙版限制作用区域。
3. 先增强中尺度结构，再少量增强高频。
4. 最后轻度降噪，不要用降噪覆盖锐化振铃。

警戒伪影：

- 行星边缘白环或黑环；
- 木星卫星出现双边；
- 土星环缝被锐化成黑线；
- 表面出现规则虫纹或颗粒；
- 极区和晨昏线出现假边。

### 不同目标注意事项

- 木星：控制处理时间窗，必要时去旋转；保护大红斑和卫星阴影边缘。
- 土星：保护环缝和环影，避免把环边锐化成硬白线。
- 火星：保护极冠和晨昏线，不放大边缘亮环。
- 金星：紫外/红外细节不是自然 RGB，输出必须标注滤镜。

### 管线边界

当前深空管线的 DBE、去星、缩星和深空拉伸不适用于行星。行星成片应在专用工具中完成，最多使用通用图像工具做尺寸、色彩空间和输出格式转换。

### 质量检查

- RGB 边缘重合，无明显红蓝轮廓。
- 行星边缘自然，无锐化环。
- 云带、风暴和环结构可在原始优秀帧中追溯。
- 背景噪声不会沿行星边缘形成硬切口。

## 3. 月面

### 数据特点

- 高信噪、高对比，视宁度和面板一致性决定质量。
- 满月与月牙、亮区与地照的动态范围差异很大。
- 高分辨率作品常需要多面板拼接。
- 月缘和明暗交界线容易出现锐化振铃。

### 视频堆栈流程

```text
各面板视频预处理
→ 帧质量排序
→ 多点对齐和堆栈
→ 各面板统一锐化基础
→ 畸变校正
→ 全景拼接
→ 亮度与色彩匹配
→ HDR/局部对比
→ 最终锐化与降噪
```

### HDR

HDR 必须来自真实的不同曝光：

- 短曝光：保护月面高光和亮缘。
- 长曝光：记录地照和阴影细节。
- 各曝光组分别堆栈后再配准。
- 使用亮度遮罩渐进合成，避免明暗交界硬边。

单张图中已经过曝的区域不能通过曲线恢复。地照增强时要控制彩噪和固定图样噪声。

### 全景拼接

- 各面板至少保留 15%–30% 重叠。
- 拍摄期间尽量保持曝光、增益、白平衡和焦点一致。
- 拼接前统一面板尺度和基础锐化，避免一块清晰一块柔软。
- 先校正局部畸变，再做几何拼接。
- 使用低频亮度匹配消除接缝，不能模糊陨石坑结构来隐藏接缝。
- 月面边缘必须使用真实覆盖区域，禁止内容感知生成缺失面板。

### 阴影细节

- 阴影区提亮应基于长曝光或足够信噪的数据。
- 使用大尺度亮度蒙版，避免把噪声颗粒变成假地形。
- 保留阴影方向和明暗交界线的物理一致性。
- 不把所有阴影提到同一亮度；适度黑位有助于保留立体感。

### 锐化

- 根据视宁度和堆栈 FWHM 设定尺度。
- 重点增强陨石坑边缘和山脊，不增强均匀月海噪声。
- 月缘使用保护蒙版。
- 拼接后只做轻度统一锐化，主要锐化应在单面板阶段完成。

### 管线边界

不要对月面使用深空 DBE、背景中性化、去星或缩星。月面 HDR、拼接和幸运成像由外部工具完成。

### 质量检查

- 面板间没有亮度、锐度或噪声接缝。
- 月缘无白边、黑边或阶梯。
- 明暗交界线没有振铃。
- 阴影细节连续，不呈现彩色斑块或塑料纹理。
- 高频噪声没有被误认为微小陨石坑。

## 4. 宽场星野

### 数据特点

- 大型渐晕、镜头阴影和传感器平场残差明显。
- 地平光害、气辉和薄云可能与银河低频结构重叠。
- 星点数量极多，缩星和背景建模都容易过度。
- 可能包含地面前景或多面板全景。

### 基础流程

```text
匹配暗场/平场校准
→ 镜头畸变与色差校正
→ 子帧局部归一化
→ 天空配准与堆栈
→ 大型渐晕/梯度诊断
→ 必要时分层 DBE
→ 星色校准
→ 保守拉伸
→ 星点控制
→ 地面前景或全景合成
```

大型渐晕和宽场 DBE 的详细判断见 `references/linear_stage_processing.md`。

### 地面前景

地面与天空通常需要不同的配准逻辑：

- 天空层按恒星配准和堆栈。
- 地面层按固定相机坐标或单独曝光处理。
- 两层必须来自同一机位和合理时间范围。
- 树木、山体和建筑边缘使用真实遮罩合成。
- 天空移动导致的边缘空缺应裁切，不能生成缺失地景。

若前景和天空曝光时间相差很大，应明确标注时间合成。自然纪实要求较高时，避免混合不同夜晚或不同月相。

### 全景拼接

- 每块面板保留足够重叠，并固定焦距、光圈、白平衡和滤镜。
- 先完成镜头配置文件校正，再进行球面/柱面投影。
- 天空面板以星点匹配，地面面板以固定特征匹配。
- 接缝优先放在低纹理区域，避开银河核心、亮星和地平线复杂轮廓。
- 各面板先做低频背景匹配，最后统一拉伸和色调。
- 不在单块面板上独立做强曲线，否则拼接后难以统一。

### 星点处理

- 密集星场可适度缩星，但不能改变星座主星的相对层级。
- 先检查星点检测掩膜是否把银河尘埃或小星云识别为星。
- 边缘色差明显时先做光学校正，不用缩星掩盖彗差。
- 星芒和亮星光晕应单独保护。

### 管线接入

完成天空堆栈或全景拼接后：

```bash
python scripts/pipeline.py widefield_master.tif widefield_final.jpg \
  --strength adaptive \
  --target-type wide_field \
  --style widefield_punch \
  --keep-all
```

若画面包含地面前景，建议先处理纯天空层，再在外部软件中与地面层合成。主管线的 DBE 和拉伸不应直接作用于地面。

### 质量检查

- 四角无明显渐晕反转或黑坑。
- 银河、气辉和真实 Hα 大尺度结构未被 DBE 清除。
- 地平线无亮边、暗边或蒙版光晕。
- 全景接缝处星点不重复、不拉长。
- 面板之间背景、星色和噪声一致。

## 5. 超新星残骸

### 数据特点

- 常见极暗、低表面亮度的丝状壳层。
- Hα、OIII 和 SII 的空间分布可能完全不同。
- 强星场会淹没细丝，去星和缩星有价值但风险较高。
- 大尺度壳层容易被背景提取误判为梯度。

代表目标：

- M1 蟹状星云：亮度较高、内部丝状结构丰富、中心脉冲星区域动态范围大。
- 面纱星云：OIII/Hα 丝状结构明显，适合 HOO。
- Simeis 147、G65.3+5.7：极暗且覆盖范围巨大，DBE 风险很高。

### 数据准备

- 优先使用窄带 Hα、OIII，条件允许时加入 SII。
- 每个通道分别校准、局部归一化、配准和堆栈。
- 通道之间必须严格星点对齐。
- 极暗目标应优先增加总曝光，而不是依赖更强后期。
- 宽场残骸需要高质量平场，减少后续 DBE 对真实壳层的伤害。

### 窄带增强策略

#### HOO

适合 Hα 与 OIII 对比强烈的残骸：

```text
R = Hα
G = 0.15 × Hα + 0.85 × OIII
B = OIII
```

比例应根据实测信噪调整，不固定追求蓝红均衡。OIII 很弱时，强行归一化会把噪声染成蓝色。

#### SHO

SII 信号充足时可用于分离冲击区：

```text
R = SII
G = Hα
B = OIII
```

必须保留各通道真实空间差异。若 SII 信噪过低，应降低其权重或放弃 SHO，而不是强降噪后硬拉。

#### RGB + 窄带

- Hα 可增强红通道中的丝状发射。
- OIII 可增强青蓝结构。
- 使用亮度或结构蒙版局部注入，避免改变全部恒星颜色。
- 星点优先保留 RGB 星色，窄带主要作用于无星目标层。

详细通道合成见 `references/narrowband_synthesis.md`。

### 极暗信号处理

推荐顺序：

```text
保守背景处理
→ 线性降噪
→ 星点分离
→ 无星层强拉伸
→ 多尺度丝状结构增强
→ 星点层独立拉伸
→ 重组
→ 最终颜色和高光控制
```

- 在线性阶段使用保守降噪，避免拉伸后噪声爆发。
- 无星层可使用更强拉伸，但必须监控背景斑驳。
- 细丝增强应使用多尺度蒙版，不能用全图锐化制造硬边。
- 星点层保持较低拉伸，避免密集星场遮盖残骸。
- 不追求把所有背景都拉到可见；没有可靠信噪的区域应保持克制。

### 背景与 DBE

- 超新星残骸可能占据整个画面，自动背景样本容易落入真实壳层。
- 宽场 Hα/OIII 数据优先检查背景模型差分。
- 诊断为无明显梯度时直接跳过 DBE。
- 必须处理时，使用低阶、大尺度模型，并排除已知壳层区域。
- 通道梯度方向不同则分别审查，不能直接全图灰度世界中和。

### 管线接入

主管线尚未接受 `supernova_remnant` 作为 `--target-type`，因此先使用其最接近的发射星云安全规则，再显式选择残骸风格：

```bash
python scripts/pipeline.py snr_combined.tif snr_final.jpg \
  --strength adaptive \
  --target-type emission_nebula \
  --color-mode emission \
  --style supernova_remnant \
  --steps color,pre_denoise,star_remove,stretch,star_process,final_color,style,star_combine \
  --keep-all
```

如果内置去星置信度低或误伤细丝，使用外部 StarNet++ 无星图并检查差分，不能接受有明显丝状结构被删除的星点层。

### 质量检查

- 丝状结构能在原始通道中追溯。
- OIII 蓝青区域不是被噪声主导。
- 星点层中没有大量残骸细丝。
- 壳层连续，不因局部增强出现断裂或黑边。
- 背景模型不包含完整弧线或冲击前沿。
- M1 等亮目标的核心不过曝，外围不被 HDR 压平。

## 统一质量审查表

| 目标 | 首要检查 | 常见失败 |
|---|---|---|
| 彗星 | 双重对齐、彗尾连续、彗核高光 | 双星、重复彗核、DBE 减尾 |
| 行星 | 帧选择、RGB 对齐、边缘自然 | 锐化环、虫纹、通道彩边 |
| 月面 | 拼接一致、HDR 来源、月缘保护 | 接缝、振铃、假阴影细节 |
| 宽场星野 | 渐晕、地平线、面板一致性 | DBE 误减银河、地景光晕、重复星 |
| 超新星残骸 | 极暗信号可信、通道比例、丝状保护 | 蓝色噪声、断丝、过度去星 |

## 明确禁止

- 为彗尾、行星云带、月面地形或残骸丝状结构补画内容。
- 对行星/月面使用生成式超分辨率。
- 把不同时刻、不同经纬结构无配准直接叠加。
