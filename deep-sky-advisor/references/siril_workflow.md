# Siril 天文后期处理工作流程

## 简介
Siril 是免费开源的天文图像处理软件，适合深空天文摄影的预处理和基础后期。

## 核心处理流程

### 1. 图像校准（Pre-processing）

#### 1.1 导入原始文件
```
文件 → 打开文件 → 选择lights、darks、flats、biases
或拖拽文件夹到 Siril
```

#### 1.2 转换文件格式
```
如果原始文件不是 FITS：
转换 → 选择文件 → 转换为 FITS
推荐设置：
- 输出格式：FITS
- 去拜耳模式：根据相机选择（OLEDCF/Mono）
```

#### 1.3 校准叠加
```
预处理 → 序列合成
步骤：
1. 选择工作目录
2. 设置校准帧：
   - Dark frames: 暗场文件
   - Flat frames: 平场文件
   - Bias/Offset: 偏置帧
3. 点击"开始预处理"
4. 输出：pp_light_*.fits（已校准的亮场）
```

### 2. 图像注册和叠加（Registration & Stacking）

#### 2.1 星点注册
```
注册 → 星点注册
设置：
- 算法：全局星点注册（推荐）或局部星点注册
- 匹配阈值：通常 10-20
- 最大变形：根据视场大小调整
```

#### 2.2 图像叠加
```
叠加 → 叠加图像
叠加类型选择：
- 中位值叠加：去异常值效果好
- 平均值叠加：SNR 最佳
- 自动排斥：自动去除异常帧

设置原则：
- 归一化方式按序列类型、天空背景变化和目标信号选择
- 拒绝算法取决于有效帧数，不固定使用同一组 sigma
- 叠加后检查 rejection map，确认异常像素、卫星轨迹被拒绝，而真实星核和目标结构未被删除
```

### 3. 后期处理（Post-processing）

#### 3.1 背景提取
```
图像处理 → 背景提取
参数原则：
- 样本必须避开目标、暗尘埃、IFN、星系外晕和亮星光晕
- 从低阶、稀疏且可靠的背景模型开始
- 发射星云和宽场银河图像不得把大尺度真实信号当作光害梯度
- 比较校正前后图像及背景模型；模型包含目标轮廓时立即回退
```

#### 3.2 色彩校准
```
图像处理 → 色彩校准
方法选择：
- 宽带 RGB/OSC 优先使用带星表和 WCS 约束的光度色彩校准
- 无法求解时先确认图像中心、像元尺寸、焦距和星点检测条件
- 窄带/双窄带数据不应套用宽带背景中性和恒星颜色假设
```

#### 3.3 去噪
```
滤镜 → 小波变换去噪
设置原则：
- 优先在线性阶段估计噪声并使用蒙版保护高信号结构
- 层数和阈值按像素尺度、噪声尺度和预览结果确定
- 处理后检查暗弱恒星、星云细丝和尘埃纹理是否被抹除
```

#### 3.4 拉伸和增强
```
色彩 → 饱和度调整
- 饱和度：+10% 到 +30%

图像处理 → 直方图拉伸
- 自动拉伸只用于预览时，不要把预览状态误认为数据已经拉伸
- 根据目标动态范围选择 Asinh、GHS 或直方图变换
- 分多次拉伸并检查黑位、高光、星色和核心结构
```

### 4. 输出
```
保存 → 保存为 FITS（保留所有数据）
导出 → 导出为 PNG/TIFF（用于分享或进一步 PS 处理）
```

## 常见问题处理

### 梯度问题
```
1. 背景提取（上述 3.1）
2. 如果还有梯度：使用"梯度消除"工具
3. 手动绘制梯度区域进行校正
```

### 星点变形
```
1. 检查单帧星形、配准和叠加前后的变化
2. 判断是跟踪、光学边场、倾斜、色差还是注册问题
3. 不要用提高 rejection 阈值或后期“修复星点”掩盖采集问题
```

### 色彩不平衡
```
1. 重新进行色彩校准
2. 手动调整色彩平衡（色彩 → 色彩平衡）
3. 检查平场、背景梯度、通道归一化和滤镜元数据，不要用中值叠加作为色彩校正方法
```

## 命令行自动化

Siril 支持脚本自动化，可以批量处理：

```bash
# 创建 Siril 脚本文件（.ssf）
# 示例：自动预处理和叠加
requires 2.0.0
load Darks
convert Darks dark
load Flats
convert Flats flat
load Bias
convert Bias bias
load Lights
convert Lights light
preprocess light -opt -debayer
register pp_light
stack r_pp_light -nonorm -out=stacked
```

## 输出文件命名规则
- `pp_light_*.fits`: 预处理后的亮场
- `r_pp_light_*.fits`: 注册后的图像
- `stacked.fits`: 最终叠加结果
- `result_*`: 各种处理的中间结果

## 与 PixInsight/PS 的衔接
Siril 处理后的文件可以：
1. 直接保存为 16-bit/32-bit TIFF 导入 PixInsight
2. 导出为 PNG 在 Photoshop 中继续处理
3. 保留 FITS 格式进行进一步天文专用处理
